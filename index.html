Experimental Results

One agent. Many real-time applications.

The agent receives only the application and a GPU budget — no hints, no templates. It self-discovers streaming, disaggregation, sequence parallelism, and pipeline parallelism. Hardware: a single node of 8× NVIDIA B200. Agent: Claude Opus 4.7 in Claude Code.

Face-to-Face Conversational Agent

ASR → LLM → streaming TTS → audio-to-video avatar. The agent composes chunk-level streaming, disaggregation of TTS and S2V engines, and pipeline parallelism inside S2V — collapsing a 98s offline baseline to a 1.43s, 21 FPS interactive stream.

70× lower latency and 21 FPS sustained throughput — vs. a hand-written single-GPU reference.

Qwen3-Omni — beating a hand-engineered baseline

A natively multimodal model: Thinker LLM → Talker LLM → Vocoder. vLLM-Omni implements disaggregation and inter-stage streaming for this workload by hand. FlashRT, given only a synchronous baseline, rediscovers the same structure — and improves on it.

Deployment GPUs Latency ↓ RTF < 1
Sequential (no streaming) 1 34.22 s ✓
vLLM-Omni (hand-engineered) 2 0.34 s ✓
FlashRT 2 0.26 s ✓

31% lower latency than vLLM-Omni's hand-engineered deployment, while keeping real-time factor below 1.

WorldPlay — flexible latency / throughput tradeoffs

An autoregressive DiT + streaming VAE video world model. The agent chooses between disaggregating DiT and VAE (for throughput via pipelining) and co-locating them with sequence parallelism (for latency) — and scales the strategy with the GPU budget.

GPUs Deployment Latency ↓ Throughput ↑
1 Baseline (sequential) 845.3 ms 18.85 FPS
2 FlashRT — throughput-optimized 884.7 ms 24.52 FPS
2 FlashRT — latency-optimized 662.1 ms 24.14 FPS
4 FlashRT — throughput-optimized 694.5 ms 31.66 FPS
4 FlashRT — latency-optimized 557.7 ms 28.75 FPS
8 FlashRT — combined 608.7 ms 37.77 FPS

~2× throughput and ~40% lower latency compared to the sequential baseline.

Krea-Realtime + SAM 3 — concurrent multi-model pipeline

A live webcam stream is split into two paths: Krea-Realtime restyles the frame, SAM 3 segments the user's body, and the outputs are composed so only the background changes. The agent automatically places SAM 3 on its own GPU and parallelizes the rest.

Deployment GPUs Latency ↓ Throughput ↑
Baseline (sequential) 1 1683.9 ms 5.85 FPS

2.2× lower latency and 2.6× higher throughput compared to the sequential baseline.

Our two visions

Generative AI in the loop. Agents in the stack.

Two research bets, one mission: keep humans at the center of creativity, and let agents take over the parts of the stack that should never have been manual.

01

Enhancing human creativity and fulfillment

We bring generative AI into the interactive, multi-sensory loop where people actually work and create — streaming text, image, audio, and video models with sub-second latency, unified multimodal architectures, and collaboration interfaces that enhance creativity and fulfillment rather than replace them.

  • Streaming, low-latency generation across every modality
  • Unified multimodal architectures, not stitched-together pipelines
  • Interfaces that keep humans in the creative driver's seat
02

Making agents first-class operators of the AI stack

We prepare the infrastructure for a world where AI agents build, deploy, optimize, secure, and evolve ML systems with minimal human intervention — and remain safe, verifiable, and robust as models and tools co-evolve alongside them.

  • Agent-authored deployment, kernels, and parallelism
  • Self-driven validation against ground-truth references
  • Verifiable, measurement-gated optimization loops
The system

An agent that deploys real-time multimodal applications.

You write a simple single-GPU reference. FlashRT lifts it into an optimized multi-GPU deployment — making its own decisions about placement, streaming, and parallelism inside a measurement-gated loop.

Today, every new real-time multimodal application — a world model, a live avatar, a speech agent — comes with its own pile of bespoke engineering: deciding what to disaggregate, what to stream, how to shard, where to place each model.

FlashRT collapses that effort into a single agent. The developer writes a simple, single-GPU reference for the application. The agent does the rest — making and verifying decisions about placement, streaming, and parallelism inside a measurement-gated loop.

The result is one serving system that can target highly diverse applications, with no per-application code on the system side.

Insight 1

Hierarchical planning

Agents can't translate a reference into an efficient deployment in one shot. Forcing them to first produce an IR, then analyze it, then lower it into a deployment dramatically improves quality — mirroring how modern compilers work.

Insight 2

Self-driven validation loop

Naive prompting yields broken code and a shallow search. A loop where the agent proposes a hypothesis, verifies it against the baseline, then re-plans from measurements produces stronger guarantees and deeper exploration.

Paper · arXiv 2026

FlashRT: Agentic System for Deploying Real-Time Multimodal Applications

Krish Agarwal1, Zhuoming Chen1, Atri Rudra2, Beidi Chen1

1Carnegie Mellon University  ·  2University at Buffalo

Real-time multimodal applications — voice agents, interactive video generation — compose heterogeneous models into pipelines whose efficient deployment demands application-specific decisions about placement, streaming, and intra-model parallelism. Existing serving systems and auto-parallelism compilers commit to limited transformation sets and fixed assumptions, so reaching high performance on a new pipeline still means hand-crafting it. FlashRT directs a generic coding agent through a multi-pass workflow — lift to an IR, validate against a reference, analyze, then iterate inside a measurement-gated loop. Across interactive video world models and multimodal LLMs, FlashRT delivers up to 70× lower latency and 2.6× higher throughput, enabling — for the first time — efficient deployment of highly diverse applications within a single serving system.
